# <a name="guide-to-enhancing-privacy-and-addressing-gdpr-requirements-with-the-microsoft-sql-platform"></a>A guia Privacidade melhora e endereçamento GDPR requisitos com a plataforma do Microsoft SQL


## <a name="summary"></a>Resumo
Em 25 de maio de 2018 uma lei de privacidade Europeu é devido a entrar em vigor que define uma nova barra global de direitos de privacidade, segurança e conformidade. Regulamentação de proteção de dados gerais ou GDPR, é basicamente sobre proteção e habilitar os direitos de privacidade dos indivíduos e estabelece os requisitos de privacidade global rígidas que governam dados pessoais como são gerenciados e protegidos, enquanto respeita escolha individual. 

Os clientes do Microsoft SQL que estão sujeitos a GDPR, se o gerenciamento baseado em nuvem ou bancos de dados local ou ambos, será necessário garantir que qualificando os dados em seus sistemas de banco de dados é adequadamente tratado e protegidos de acordo com os princípios GDPR. Isso significa que muitos clientes precisará revisar ou modificar o gerenciamento de banco de dados e procedimentos, manipulação de dados especialmente voltados para a segurança de processamento de dados como estipulados no GDPR.

Tecnologias da Microsoft com base em SQL oferecem vários recursos de segurança internas que podem ajudar a reduzir os riscos para dados e melhorar a proteção e a capacidade de gerenciamento de dados no nível do banco de dados e depois. Este artigo examina esses recursos e compartilha algumas abordagens da Microsoft usando o Microsoft SQL para atingir as metas de privacidade de dados de GDPR.
