# Script de tarefas administrativas no Analysis Services
  Você pode automatizar tarefas administrativas do [!INCLUDE[ssASnoversion](../../includes/ssasnoversion-md.md)] gravando ou gerando scripts que podem ser executados manualmente ou agendados pelo SQL Server Agent. A tabela a seguir resume as opções de script disponíveis para você e fornece links para mais informações.  
  
 Todas as metodologias listadas abaixo oferecem suporte a scripts que podem ser salvos a um arquivo e executados como uma operação independente. Como a linguagem DAX (Data Analysis Expression) usada para os modelos de tabela e as pastas de trabalho do [!INCLUDE[ssGemini](../../includes/ssgemini-md.md)] não atendem aos critérios, ela não está incluída na lista a seguir.  
  
|Metodologia|Formato do arquivo|Description|Links|  
|-----------------|-----------------|-----------------|-----------|  
|PowerShell|.ps1|O Analysis Services dá suporte ao ambiente de script PowerShell do SQL Server por meio de um provedor SQLAS que adiciona a navegação de objetos por meio da linha de comando, bem como cmdlets para tarefas administrativas, como backup, restauração, processamento e gerenciamento de funções.<br /><br /> Além disso, o provedor inclui um cmdlet de uso geral, **Invoke-ASCmd**, que permite executar os arquivos de script TMSL, ASSL-XMLA, MDX ou DMX em uma sessão do PowerShell.<br /><br /> O script do PowerShell do Analysis Services é compatível com modelos multidimensionais e tabulares, mas não servem para as pastas de trabalho [!INCLUDE[ssGemini](../../includes/ssgemini-md.md)] acessadas a partir do SharePoint.|[PowerShell scripting in Analysis Services](../../analysis-services/instances/powershell-scripting-in-analysis-services.md)|  
|TMSL|.json|TMSL (Linguagem de Scripts de Modelo de Tabela) é uma extensão do XMLA que fornece uma interface de comando e definições de objeto para modelos de Tabela e bancos de dados em um servidor SQL Server 2016.<br /><br /> Escolha esta abordagem quando seus requisitos incluírem um dos seguintes:<br /><br /> – O modelo ou o banco de dados é Tabela no nível de compatibilidade 1200.<br />– Seu script cria objetos diretamente em um servidor ou executa as tarefas de definição de dados e operacionais (por exemplo, recriar e processar o banco de dados).<br />– A reutilização em várias ferramentas e tecnologias é uma prioridade. Os scripts TMSL podem ser adicionados a tarefas de comandos do Analysis Services no SQL Server Agent, referenciadas em pacotes SSIS ou em scripts do PowerShell.<br />– O script deve ser executado de forma autônoma. Você pode usar o SQL Server Agent para agendar um trabalho que contém script TMSL ou um pacote SSIS que contém XMLA.<br />– O XMLA é um requisito de aplicativo. XMLA é uma interface que não requer um ambiente de código gerenciado. Você pode executar script XMLA em um aplicativo que não usa o .NET Framework.|[Referência de TMSL &#40;Linguagem de Scripts de Modelo de Tabela&#41;](../../analysis-services/tabular-model-scripting-language-tmsl-reference.md)<br /><br /> [PowerShell scripting in Analysis Services](../../analysis-services/instances/powershell-scripting-in-analysis-services.md)<br /><br /> [Nível de compatibilidade para modelos de tabela no Analysis Services](../../analysis-services/tabular-models/compatibility-level-for-tabular-models-in-analysis-services.md)|  
|Script ASSL ou XMLA|.xmla|ASSL (Linguagem de Scripts do Analysis Services) é uma extensão do XMLA que fornece acesso a dados de objetos e operações em uma instância do Analysis Services executada no modo de tabela ou multidimensional. A linguagem ASSL inclui definição de dados e suporte de linguagem de comandos, permitindo que a expressão completa de objetos e operações do Analysis Services em um formato XML. Os scripts que usam os objetos e comandos fornecidos por ASSL são salvos como arquivos .xmla. Dentro do contexto do Analysis Services, é uma prática comum para recorrer à linguagem ASSL como script XMLA.<br /><br /> Escolha esta abordagem quando seus requisitos incluírem um dos seguintes:<br /><br /> – O modelo ou o banco de dados é Multidimensional ou Tabela em níveis de compatibilidade inferiores (1050, 1100, 1103).<br />– Seu script cria objetos diretamente em um servidor ou executa as tarefas de definição de dados e operacionais (por exemplo, recriar e processar o banco de dados).<br />– A reutilização em várias ferramentas e tecnologias é uma prioridade. Scripts XMLA podem ser adicionados a tarefas de comandos do Analysis Services no SQL Server Agent, referenciadas em pacotes do SSIS ou em scripts do PowerShell.<br />– O script deve ser executado de forma autônoma. Você pode usar o SQL Server Agent para agendar um trabalho que contém script XMLA ou um pacote do SSIS que contém XMLA.<br />– O XMLA é um requisito de aplicativo. XMLA é uma interface que não requer um ambiente de código gerenciado. Você pode executar script XMLA em um aplicativo que não usa o .NET Framework.|[Criar scripts do Analysis Services no Management Studio](../../analysis-services/instances/create-analysis-services-scripts-in-management-studio.md)<br /><br /> [Usar modelos do Analysis Services no SQL Server Management Studio](../../analysis-services/instances/use-analysis-services-templates-in-sql-server-management-studio.md)<br /><br /> [Agendar tarefas administrativas do SSAS com o SQL Server Agent](../../analysis-services/instances/schedule-ssas-administrative-tasks-with-sql-server-agent.md)<br /><br /> [Desenvolvendo com ASSL &#40;linguagem de script do Analysis Services&#41;](../../analysis-services/multidimensional-models/scripting-language-assl/developing-with-analysis-services-scripting-language-assl.md)<br /><br /> [Cmdlet Invoke-ASCmd](../../analysis-services/powershell/invoke-ascmd-cmdlet.md)|

No nível de comando, como para processar, fazer backup ou restaurar, design de agregação ou outro comando, você pode gerar script usando o recurso Script na caixa de diálogo, escolhendo opções que colocam o script em uma nova janela, arquivo ou área de transferência. Você também pode gravar scripts XMLA manualmente em um editor de texto ou de códigos, ou usar um modelo do Gerenciador de Modelos.<br /><br /> Para executar o script, use uma destas abordagens:<br /><br /> – Use o Management Studio para criar ou modificar diretamente objetos em uma instância do [!INCLUDE[ssASnoversion](../../includes/ssasnoversion-md.md)].<br />– Use o SQL Server Agent para agendar um trabalho que inclui uma tarefa de comando do Analysis Services.<br />– Use o cmdlet Invoke-ASCmd para executar o script em uma sessão do PowerShell. |MDX Script|.mdx|A linguagem de expressão MDX é uma consulta padrão do setor e uma linguagem de expressão para fontes de dados analíticas que também faz parte da especificação XMLA.<br /><br /> Você pode criar um arquivo de script MDX autônomo que consulte dados ou informações do sistema. Por exemplo, as exibições DMV (exibição de gerenciamento dinâmico) que expõem informações sobre as operações do servidor local e a integridade do servidor são acessadas por meio da instrução MDX Select.<br /><br /> O script MDX será executado em servidores no modo Multidimensional e de Tabela anteriores. Você pode executar o script de forma interativa no SQL Server Management Studio ou em uma sessão do PowerShell usando **Invoke-ASCmd**.|[Conceitos básicos do script MDX &#40;Analysis Services&#41;](../../analysis-services/multidimensional-models/mdx/mdx-scripting-fundamentals-analysis-services.md)<br /><br /> [Usar DMVs &#40;Exibições de Gerenciamento Dinâmico&#41; para monitorar o Analysis Services](../../analysis-services/instances/use-dynamic-management-views-dmvs-to-monitor-analysis-services.md)<br /><br /> [Usar modelos do Analysis Services no SQL Server Management Studio](../../analysis-services/instances/use-analysis-services-templates-in-sql-server-management-studio.md)|
